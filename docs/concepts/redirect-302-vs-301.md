# 短链跳转：302 vs 301 vs 308 的工程取舍

> **5 分钟标尺**：能讲清三个状态码的语义差异、为什么短链要用 302、什么时候反而该用 301。  
> **slink 决策**：默认 **302 Found**。位置 `internal/api/redirect.go`。

---

## 一、HTTP 重定向状态码三件套

| 状态码 | 名称 | 浏览器/CDN 默认行为 | 客户端缓存 |
|---|---|---|---|
| 301 | Moved Permanently | "永久搬走了" | **强缓存**，下次直接打新地址 |
| 302 | Found | "暂时去那边" | 不缓存（默认） |
| 308 | Permanent Redirect | 301 的 method-preserving 版 | 强缓存 |
| 307 | Temporary Redirect | 302 的 method-preserving 版 | 不缓存 |

**method-preserving** 是什么：301/302 历史上有个坑——浏览器有时把 POST 自动转成 GET 跟随，违反 RFC。  
307/308 强制保留原 method。短链都是 GET，这点用不上。

slink 的实际选项：**302 vs 301**。

---

## 二、短链业务为什么默认 302

短链有一个核心需求：**每次点击都要打到服务端**（采集事件、做风控、做计费）。

```
用户点 https://o.cn/Abc123
  ↓
DNS / TLS / TCP
  ↓
GET /Abc123 → slink server
  ↓
Server 记录"点了 Abc123" + 返回 302 Location: https://商家.com/landing
  ↓
浏览器跳走
```

如果用 **301 永久跳转**：

```
第 1 次：浏览器 → slink → 拿到 301 + Location → 浏览器记入"永久缓存"
第 2 次：浏览器 → 直接打 https://商家.com/landing  ← slink 完全看不到这次点击
第 3 次：同上...
```

**事件采集断了**，营销分析、风控、A/B 测试、改 long_url 全部失效。

---

## 三、CDN 也有缓存

不只浏览器。中间所有 CDN/反向代理（Cloudflare、阿里云 CDN、企业内网代理）默认都会缓存 301，不会缓存 302。

短链业务上 CDN 加速静态资源是常见做法，CDN 看到 301 后就把这个跳转关系**全 PoP 节点缓存**——后果同上但放大 N 倍。

---

## 四、那 301 什么时候用？

仅当满足两个条件：

1. **跳转关系永远不会改**（long_url 不会更新）
2. **不需要采集每次点击**（纯导航类、不做营销/风控）

例子：旧域名整站 301 到新域名。但短链场景几乎不满足条件 1——商家就指望"链接不变，落地页可换"这个能力来做 A/B 测试和热修。

---

## 五、隐含成本对比

| 维度 | 301 | 302 |
|---|---|---|
| 服务端 QPS | 极低（首次后客户端不再访问） | 全量（每次点击都打过来） |
| 事件采集 | ❌ 断 | ✅ 全 |
| long_url 改动生效 | 等浏览器缓存过期（可能数小时） | 立即生效 |
| 风控 / 反作弊 | ❌ 看不到点击 | ✅ 实时监控 |
| 服务器成本 | 低 | 高（要扛住所有点击 QPS） |

短链业务里"服务器成本高"反而是好事——QPS 越高越证明业务价值。slink v0.1 的 5w QPS 目标就是为这个准备的。

## 六、slink 的实现

```go
// internal/api/redirect.go
http.Redirect(w, r, link.LongURL, http.StatusFound)  // 302
```

关键细节：

- **不用 `http.StatusMovedPermanently`（301）** 是有意决策，不是默认值
- **不用 `http.StatusTemporaryRedirect`（307）** 因为 GET 下 302 和 307 行为一致，302 兼容性更好（老客户端）
- 跳转响应有 HTML body（`<a href="...">Found</a>.`）—— Go 标准库强加，无法去掉除非自己写 ResponseWriter；body 极小不影响性能

---

## 七、特殊状态码：410 Gone

slink 还用了 410：

```go
if link.IsExpired(time.Now()) {
    writeError(w, http.StatusGone, ErrNotFound, "link expired")
    return
}
```

**410 vs 404 的语义区别**：

- 404 Not Found："这个东西从来没存在过 / 我不知道"
- 410 Gone："这个东西曾经在，现在永远没了"

短链过期是后者：商家创建过、TTL 到了、永远不会再生效。  
搜索引擎收录时如果碰到 410 会**主动删掉索引**；404 会保留一段时间。  
对用户：410 让客户端能区分"是不是过期了" vs "是不是写错链接"。

---

## 八、长期看：是否要"动态选择"

成熟系统会按"短链类型"动态选状态码：

- 营销链接（要事件采集）→ 302
- 客户已弃用的旧链接（永久转向新链接）→ 301
- 已过期 → 410

slink v0.1 全部 302，简单可控。v0.3 加 link.Type 字段后再扩。
